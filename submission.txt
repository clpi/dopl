PR Title: ⚡ Optimize esc_str string escaping function in parser

💡 **What:**
The `esc_str` function was rewritten to use bulk copy functions instead of copying byte-by-byte in a `for` loop. The implementation now relies on `strchr(p, '\\')` to fast-forward to the next escape character, and copies the standard, un-escaped text in blocks utilizing highly optimized `memcpy` and `strcpy` implementations provided by the system's standard library.

🎯 **Why:**
The previous method of parsing strings evaluated the string buffer byte-by-byte checking if the character is an escape sequence `\\`. While this is functional, on larger strings that are mostly comprised of un-escaped standard characters this introduces unnecessary loop overhead. Using `strchr` combined with `memcpy` leverages SIMD instructions generally found in most C library implementations which yields better throughput when skipping long stretches of standard characters.

📊 **Measured Improvement:**
A focused benchmark was created to parse normal text containing multiple escapes back-to-back vs long normal texts multiple times in a loop over 1,000,000 iterations.

Baseline (byte-by-byte): 0.183422s
Optimized (strchr + memcpy): 0.124395s
Improvement: ~32% speedup in raw execution time during the microbenchmark.
